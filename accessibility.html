Accessibility Statement

Last reviewed: 1 May 2025 · NexScan GmbH

WCAG 2.2 Level AA – Partially Conformant We are committed to meeting all Level AA success criteria. Known exceptions are listed below.

1. Commitment to accessibility

NexScan GmbH is committed to making the DevScan website accessible to as many people as possible, regardless of technology or ability. We aim to meet the Web Content Accessibility Guidelines (WCAG) 2.2 at Level AA, published by the World Wide Web Consortium (W3C).

These guidelines explain how to make web content more accessible to people with disabilities, including users who rely on screen readers, keyboard navigation, or other assistive technologies.

2. Measures taken

We have implemented the following accessibility measures across all pages of this website:

Perceivable

  • All non-text content (icons, graphics, SVG illustrations) has a text alternative via alt attributes or aria-label. Purely decorative elements are hidden from assistive technologies using aria-hidden="true".
  • Color is never used as the sole means of conveying information — all states (error, success, disabled) are also communicated via text or icons.
  • Text contrast ratios meet or exceed 4.5:1 for normal text and 3:1 for large text and UI components, verified against WCAG 2.2 Success Criterion 1.4.3.
  • Text can be resized up to 200% without loss of content or functionality.
  • No content flashes more than three times per second (no risk of photosensitive seizures).

Operable

  • All interactive elements — navigation links, buttons, form fields, and the cookie banner — are fully keyboard-operable using Tab, Enter, and Escape.
  • A visible skip-to-content link is provided at the top of every page, allowing keyboard users to bypass the navigation bar.
  • Focus indicators are visible on all interactive elements.
  • No time limits are imposed on any interaction on this website.
  • Page titles are unique and descriptive on every page.

Understandable

  • The language of each page is declared via the lang="en" attribute on the <html> element.
  • Navigation is consistent across all pages.
  • All form inputs have a visible, programmatically associated <label>. Required fields are identified both visually and via aria-required="true".
  • Error messages (e.g., in the contact form) identify the field in error and describe how to fix it.

Robust

  • HTML is written to be valid and semantic, using elements such as <nav>, <main>, <header>, <footer>, <section>, and <article> to convey document structure.
  • ARIA landmarks and roles are used where native HTML semantics are insufficient.
  • The cookie banner and mobile menu are implemented as accessible dialogs with appropriate role, aria-modal, and aria-hidden attributes.
  • The website is tested with keyboard-only navigation and the WAVE and axe DevTools browser extensions.

3. Known limitations

While we strive for full WCAG 2.2 AA conformance, the following known limitations exist. We are actively working to resolve them:

Page / Component Issue WCAG criterion Status
Contact form Inline error messages are not yet implemented; required field validation relies on the browser's native UI, which varies in accessibility across assistive technologies. 3.3.1 Error Identification (A) Planned fix: Q3 2025
Code window (hero) The code mockup is presented as a static image-like block. Although an aria-label provides a text description, the content is not exposed as structured text to screen readers. 1.1.1 Non-text Content (A) Workaround in place (aria-label); full fix planned
External links Some external links do not explicitly announce that they open in a new tab. This will be addressed by adding visible and accessible indicators. 3.2.2 On Input (A) Planned fix: Q3 2025

4. Technical specifications

The accessibility of this website relies on the following technologies:

  • HTML5
  • CSS3
  • JavaScript (ES6+)
  • WAI-ARIA 1.2

These technologies are relied upon for conformance. The website is tested in the latest versions of Chrome, Firefox, and Safari on desktop and mobile.

5. Assessment approach

NexScan GmbH assessed the accessibility of this website through the following methods:

  • Self-evaluation using the WAVE Web Accessibility Evaluation Tool
  • Automated testing using the axe DevTools browser extension
  • Manual keyboard-only navigation testing
  • Manual review against the WCAG 2.2 success criteria checklist
